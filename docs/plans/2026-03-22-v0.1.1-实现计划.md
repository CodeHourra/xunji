| 维度 | v0.1.1 方向 |

## 2. 与 v0.1 的差异边界

- **不做**：新数据源采集、MCP、自动同步后台、SSH 采集（仍归 v0.2+）。
- **可做**：本地 DB + 文件系统的导出、UI/文案、轻量 Tauri 插件（对话框、落盘）、前端样式与信息展示、队列与错误信息透传。

## 3. 导出功能（核心）

### 3.1 需求分层

| 能力 | 场景 | 建议交互 |
|------|------|----------|
| **单条 MD** | 会话详情页已有关联笔记 | 操作区增加「导出 Markdown」→ 系统 **保存文件** 对话框，默认文件名由标题安全化 + 短 id 防碰撞 |
| **勾选导出** | 知识库批量 | 列表/卡片视图支持 **多选** →「导出所选」→ **选择文件夹**，将每条笔记写为独立 `.md` |
| **全部导出** | 备份或迁移 | 「导出全部笔记」→ **选择文件夹** → 导出 **库内全部卡片**（**无视当前列表/侧栏筛选**，见 3.3） |

### 3.2 文件内容建议（单文件 Markdown）

为便于外部工具与 Git 使用，建议在正文前增加 **YAML frontmatter**（单条/批量一致）：

- `title`、`source`、`project`、`created_at`、`tags`、`card_type`、`session_id`、`card_id`
- 正文：现有 `note` 字段（已是 Markdown）

具体字段以实现时 DB 可用字段为准，并在 `docs/specs` 中补一页「导出格式说明」。

### 3.3 「全部导出」的语义（已拍板）

- **「导出全部笔记」** = **数据库中全部知识卡片**（与当前知识库页筛选条件 **无关**）。
- UI 文案需明确写出类似：**「将导出库内全部笔记（共 N 条），不受当前筛选影响」**；执行前可展示 N（一次计数查询）。
- **「导出所选」** 仍仅导出勾选条目，与「全部导出」并存，避免混淆。

### 3.4 技术要点（Tauri 2）

当前工程仅有 `core:default`，落盘与对话框需扩展能力，例如：

- 使用 **`tauri-plugin-dialog`**：`save`（单文件）、`open`（目录）。
- 使用 **`tauri-plugin-fs`** 或 **`@tauri-apps/plugin-fs`**：在用户选定路径下创建/写入文件（注意 **scope** 与权限配置，仅写入用户选择的路径）。

后端建议新增命令（名称待定）：

- `export_card_markdown`：单卡 → 单路径（优先 **Rust 读库拼内容**）。
- `export_cards_markdown_dir`：`card_ids: string[]` + 目录；**安全文件名**，冲突时自动后缀。
- `export_all_cards_markdown_dir`：仅选目录，**SQL 拉取全部卡片 id** 后批处理写出（或流式写入控制内存）。

**日志**：导出成功/失败路径、条数，便于排查权限问题。

### 3.5 验收要点

- 单条：重复导出覆盖提示（或自动加后缀）行为符合预期。
- 批量：数百条时 UI 有进度或分步反馈（可简化为 loading + 完成 Toast）。
- 全部：条数 N 与库内卡片总数一致（抽样核对）；筛选切换后「全部导出」结果不变。
- 异常：无写权限、磁盘满、非法路径有明确中文提示。

## 4. 会话与笔记：路径、ID 与列表布局

### 4.1 会话列表

| 项 | 说明 |
|----|------|
| **路径与 ID** | 在每条会话卡片上展示 **会话路径**（如 `raw_path` / `project_path` 等现有字段中最适合用户理解的一项或组合）与 **会话 id**（`session_id` 或 DB 主键，与实现一致）；过长 **省略 + Tooltip 全文**。 |
| **布局** | **项目名** 与 **时间** 同一行展示；时间使用 **`NTooltip`**，说明文案为 **「会话时间」**，内容为 **完整、可读的绝对时间**（若仅有 `updated_at`/`created_at`，在 tooltip 或副文案中注明对应字段，避免误解）。 |
| **标题文案** | 使用 **会话中首条 user 消息** 截取合适长度作为 **列表主标题**（单行省略）；**Tooltip** 展示 **更多正文**（建议上限如 300～500 字，避免 DOM 过大）。若无 user 消息，退回项目名或 session id 缩写。 |

**产品侧补充建议（实现时可择优）：**

- 首条 user 往往比会话文件名更贴近「聊了什么」，适合作主标题；若会话 **已有关联卡片** 且希望强调沉淀结果，可增加 **副标题一行**：卡片标题（小字/次要色），避免与主标题重复堆叠。
- 截断策略：按 **字符/码点** 截断并加省略号，中英文混排时优先保证不在多字节字符中间切开（可用已有工具或简单循环）。
- 路径展示：若完整路径过长，可只高亮 **最后 1～2 段路径** + Tooltip 全路径。

### 4.2 会话详情

| 项 | 说明 |
|----|------|
| **路径与 ID** | 在详情页显著区域（如 `SourceInfo` 附近或独立「元信息」行）展示 **会话路径** 与 **会话 id**；支持复制（可选 v0.1.1 或后续）。 |

### 4.3 笔记详情（含卡片视图）

| 项 | 说明 |
|----|------|
| **来源会话** | 在笔记详情中增加 **来源会话的路径** 与 **会话 id**（与 4.1 字段一致）；与现有「来源 / 项目」信息合并展示，避免重复啰嗦。 |

**数据**：若前端当前缺字段，需在 **`get_session` / `get_card` / 列表 DTO** 中补齐路径类字段（以 DB 已有列为准）。

## 5. 笔记 Markdown 阅读样式

| 项 | 说明 |
|----|------|
| **现状** | 渲染偏素，层次弱。 |
| **目标** | 在 **不引入自定义组件库替代 Markdown 渲染** 的前提下，通过 **Typography**（`@unocss/preset-typography` 或 scoped 样式）增强：标题层级、引用、列表、行内代码/代码块、表格、链接 hover；与深浅色主题一致。 |
| **注意** | 控制与全局 Naive UI 的字体与间距协调，避免「网页感」过重。 |

---

## 6. 分析队列与失败信息

| 项 | 说明 |
|----|------|
| **队列明细** | 批量任务执行过程中，用户在 **分析队列面板** 中应能 **展开或进入明细**：当前/每条任务状态、会话标题、**可点击跳转会话**（可选）、排队顺序；避免只能看到进度条而不知道在跑哪几条。 |
| **失败原因** | 分析失败时，在 **Toast / 队列条目 / 会话卡片** 至少一处展示 **后端或 Sidecar 返回的可读原因**（网络、JSON 解析、API Key、超时等）；技术细节可折叠，主因一句话可见。 |
| **实现提示** | 需串联 `distill_session` 错误字符串、Sidecar stderr、Rust `map_err` 信息，避免 `Error` 泛化丢失；前端对 `invoke` 错误做稳定展示。 |

## 7. 体验与文案（与上并行、可拆 PR）

| 项 | 说明 |
|----|------|
| **搜索** | 对话记录页搜索框：占位符与辅助说明改为明确 **「搜索知识笔记（全文）」**；无结果时区分「无笔记」与「无匹配关键词」。 |

| **关于寻迹** | 设置内 Tab「关于寻迹」：展示应用名称、应用版本、Tauri 版本、应用标识（与 `tauri.conf.json` 一致）；**分版本更新日志**（**唯一维护源** 仓库根目录 `CHANGELOG.md`，`npm run sync:changelog` 同步至 `apps/desktop/src/data/app-changelog.md`）。 |
