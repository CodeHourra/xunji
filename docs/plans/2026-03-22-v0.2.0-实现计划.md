# 寻迹 v0.2.0 实现计划

> **基线版本**：v0.1.1  
> **目标版本**：v0.2.0  
> **原则**：在 MVP + 导出与信息架构之上，按 PRD「体验完善」推进；**大项可分批合入**，允许里程碑内调整顺序。  
> **需求来源**：[`docs/PRD/寻迹（XunJi）产品需求文档 PRD.md`](../PRD/寻迹（XunJi）产品需求文档 PRD.md) §六 `v0.2 - 体验完善`  
> **任务跟踪（执行时更新状态）**：[`2026-03-22-v0.2.0-tasks.md`](./2026-03-22-v0.2.0-tasks.md)

---

## 1. PRD 中的 v0.2 范围（原文归纳）

| # | 能力 | 说明 |
|---|------|------|
| 1 | **CodeBuddy 数据采集** | 与 Claude Code / Cursor 并列的新采集管线（路径与格式见 PRD 数据源章节；含 CLI / JetBrains 等变体时需分阶段） |
| 2 | **自动分类树** | 知识或会话侧可维护的层级分类（与现有「类型 / 标签」关系需产品定界：独立树 vs 映射到 category） |
| 3 | **标签系统完善** | 标签 CRUD、批量打标、侧栏与筛选一致性、与提炼标签流贯通 |
| 4 | **统计 Dashboard** | 会话量、卡片量、来源分布、时间趋势等可视化（只读即可先行） |
| 5 | **Obsidian 导出** | 在现有 Markdown 导出之上，适配 Obsidian 目录/链接约定（如双链、frontmatter 字段约定可选） |
| 6 | **系统托盘 + 后台静默同步** | 托盘图标与菜单；定时或事件触发的增量同步（与 v0.1「手动同步」并存，可配置关闭） |

PRD 目标周期参考：**+3 周**（执行时以任务拆分为准）。

---

## 2. 与 v0.1.1 的边界

- **纳入 v0.2**：上表六项及支撑它们的配置、DB 迁移、权限与打包说明。
- **明确不纳入 v0.2**（对齐 PRD §六后续版本）：MCP Server、向量语义搜索、卡片编辑器、多 LLM 配置中心、Windows 正式支持等 —— 归 **v0.3+ / v1.0**。

---

## 3. 建议实施顺序（可并行）

以下为依赖较少的推荐顺序，实施中可按人力调整。

```text
1) CodeBuddy 采集（Rust collector + scheduler 接入 + 配置项）
      ↓
2) 标签系统完善（DB/API/UI 一体，为分类树打基础）
      ↓
3) 自动分类树（依赖 category / 树形模型与 UI 侧栏）
      ↓
4) 统计 Dashboard（只读聚合查询 + 新路由或设置内 Tab）
      ↓
5) Obsidian 导出（复用 export 命令，增加「导出模式」或独立子命令）
      ↓
6) 托盘 + 静默同步（Tauri tray、定时器/Launch Agent 文档、与现有 sync 去重）
```

**说明**：若产品希望先「看得见」的进展，可将 **Dashboard** 或 **Obsidian 导出** 提前；**托盘+后台**涉及常驻与电量策略，建议靠后或单独里程碑。

## 4. 技术要点备忘

| 主题 | 要点 |
|------|------|
| CodeBuddy | 与 `cursor` 类似可能含 SQLite / JSONL；JetBrains 路径需沙箱与权限说明 |
| 分类树 | `categories` 表已有雏形时需对齐迁移；前端 `NTree` 与知识库筛选联动 |
| 标签 | 与 `card_tags` / 提炼侧标签去重规则一致 |
| Dashboard | 聚合 SQL 或物化统计表；注意大数据量下的查询耗时 |
| Obsidian | vault 结构、附件路径、可选 wikilink |
| 托盘 | Tauri 2 tray API、macOS 菜单；静默同步与「仅前台同步」互斥配置 |
