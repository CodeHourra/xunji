# 寻迹 v0.1 功能清单

> **版本**：v0.1（MVP）  
> **说明**：本文档依据当前仓库实现梳理，与 [`docs/plans/2026-03-20-v0.1-MVP实现计划.md`](../plans/2026-03-20-v0.1-MVP实现计划.md) 对齐，并标注与 PRD 长期愿景的边界。  
> **更新日期**：2026-03-22

---

## 1. 范围说明

| 项 | 说明 |
|----|------|
| 交付形态 | Tauri 2 桌面应用（macOS 为主），本地 SQLite 存储 |
| 已实现数据源采集 | **Claude Code**（JSONL）、**Cursor**（`state.vscdb` + Lexical 文本抽取） |
| 配置中的其他数据源 | 可在 `config.toml` 中出现（如 CodeBuddy 等），侧栏可展示品牌信息；**Rust 调度器仅对 `claude-code` / `cursor` 执行采集**，其余 `source.id` 会记录告警并跳过 |
| 未纳入 v0.1 | MCP Server 对外服务、文件 Watch/定时自动同步（配置项预留）、SSH 远端采集、PRD 中 P1/P2 其他 IDE 解析器 |

---

## 2. 桌面应用壳与全局体验

| 功能 | 说明 |
|------|------|
| 主布局 | 顶栏 + 左侧边栏 + 主内容区；底部 **分析队列面板**（`AnalysisQueuePanel`） |
| 顶栏 Tab | **对话记录** ↔ **知识库**，与路由 `/`、`/library` 联动 |
| 一键同步 | 顶栏「同步」：调用 `sync_all`，刷新侧栏会话分组 |
| 主题 | 浅色 / 深色切换（Naive UI `NConfigProvider` + 本地 UI 状态） |
| 设置入口 | 顶栏打开 **设置** 弹窗（`SettingsModal`） |
| 导出按钮 | 顶栏「导出」图标：**当前为占位**，无绑定业务逻辑 |

---

## 3. 数据采集

| 功能 | 说明 |
|------|------|
| 手动全量同步 | `CollectorScheduler::collect_all()`：按配置遍历已启用数据源，汇总 `SyncResult`（found / new / updated / skipped） |
| Claude Code | 扫描配置的 `scan_dirs`，解析 `projects/.../*.jsonl`；归一化为 `NormalizedSession` / `NormalizedMessage` |
| Cursor | 只读打开 `.vscdb`（SQLite），解析对话相关键值与 Lexical 结构，抽取纯文本气泡 |
| 去重与增量 | Session 级去重；消息数变化时标记更新（与列表「待重新分析」等业务状态配合） |
| 同步日志 | 写入 `sync_log`（详见存储层） |

---

## 4. 本地存储与检索

| 功能 | 说明 |
|------|------|
| 数据库 | SQLite，默认路径 `~/.xunji/db/xunji.db`，WAL、迁移版本管理 |
| 实体 | 会话、消息、知识卡片、标签、分类、同步日志、Token 使用等（以 `storage/migrations.rs` 为准） |
| 会话列表查询 | 分页、`source` / `host` / `project` / `status` 等筛选（`list_sessions`） |
| 知识卡片 | 分页列表、按标签 / 类型 / 价值等筛选（`list_cards`）；单卡详情（`get_card`） |
| 全文搜索 | FTS5 搜索卡片（`search_cards`），支持标签、卡片类型过滤 |

---

## 5. LLM 提炼（Sidecar）

| 功能 | 说明 |
|------|------|
| 进程模型 | Bun 构建的 Sidecar，与 Rust 通过 **stdin/stdout 逐行 JSON-RPC 2.0** 通信 |
| JSON-RPC 方法 | `ping`、`init`（注入 API/模型配置）、`judge_value`（轻量价值判断）、`distill_full`（完整笔记 JSON） |
| 文本预处理 | 清洗无关标记、截断过长内容（见 `packages/sidecar/src/distiller/preprocessor.ts`） |
| API 模式 | OpenAI-compatible HTTP（`openai` SDK），支持多厂商预设（设置中可选 provider / baseUrl / model / 超时） |
| CLI 模式 | 通过配置指定本地 CLI 命令与参数，由 Sidecar 以子进程方式调用（与 API 模式二选一，由 `distiller.mode` 控制） |

---

## 6. 会话提炼流水线（Rust + Sidecar）

| 功能 | 说明 |
|------|------|
| 命令 | `distill_session`：读取会话消息 → 拼接正文 → Sidecar `init` → `judge_value` |
| 分支逻辑 | 价值为 medium/high 时继续 `distill_full` 并写入卡片；低价值或无价值时更新会话元信息，可能不生成卡片（前端通过 `DistillSessionResult` 区分） |
| 追踪 | 支持 `traceId` 等日志字段，便于排查单次提炼问题 |
| Sidecar 缺失 | 若未找到 Sidecar 二进制，启动时告警，提炼不可用 |

---

## 7. 前端：对话记录（`/）

| 功能 | 说明 |
|------|------|
| 工具栏 | 搜索框（**卡片 FTS**，300ms debounce）、刷新、**已分析 / 待分析** 快捷筛选 |
| 会话列表 | `SessionCard` 展示摘要、价值、状态等；分页 |
| 搜索模式 | 输入关键词后切换为 **搜索结果列表**（卡片摘要），点击进入详情 |
| 单条分析 | 未分析会话可发起提炼（走全局分析队列） |
| 批量分析 | 批量选择待分析会话，**串行**入队执行（与 Sidecar 单通道一致） |
| 空状态 / 加载 | 无数据时引导「立即同步」；列表加载中 Spin |

---

## 8. 前端：知识库（`/library`）

| 功能 | 说明 |
|------|------|
| 卡片列表 | `list_cards` 分页，侧栏筛选（标签、类型、技术栈等，与 `filters` / 侧栏 store 联动） |
| 视图模式 | 列表 / 卡片网格，**视图偏好持久化**（localStorage） |
| 跳转详情 | 点击卡片进入会话详情（带 `cardId` 查询参数） |

---

## 9. 前端：会话详情（`/session/:sessionId`）

| 功能 | 说明 |
|------|------|
| 笔记展示 | Markdown 渲染（`marked` + `highlight.js`） |
| 对话回放 | `ChatReplay`：气泡展示 user/assistant；思考块等可折叠（以组件实现为准） |
| 来源与消耗 | `SourceInfo`：来源、项目、日期、模型与 Token 等展示 |
| 操作栏 | `ActionBar`：**查看笔记 / 查看对话** 切换、**重新分析**、关闭 |
| 与队列联动 | 重新分析等可走全局队列，避免并发冲突 |

---

## 10. 左侧边栏

| 功能 | 说明 |
|------|------|
| 目录树 | **数据源 → 主机 → 项目** 三级结构，展示各节点会话数量 |
| 筛选联动 | 点击树节点更新 `filters` 并刷新会话列表 |
| 品牌展示 | Claude Code、Cursor、CodeBuddy 等品牌图标与标签（与配置中的 `source.id` 对应） |

---

## 11. 设置

| 功能 | 说明 |
|------|------|
| 配置读写 | `get_config` / `save_config`，持久化 `~/.xunji/config.toml`，保存后内存热更新 |
| 提炼引擎 | **API 模式** 与 **CLI 模式** 切换及对应字段编辑 |
| 数据源 | 各数据源的启用开关与扫描路径（展示与保存；**实际采集仍仅 claude-code / cursor**） |
| CLI 自动检测 | `probe_cli_tools`：在登录 shell 环境下解析常见 AI CLI 路径，便于填写 CLI 命令 |

---

## 12. 全局分析队列

| 功能 | 说明 |
|------|------|
| 串行执行 | 多条提炼任务排队，**同时仅一条** `distill_session` 执行 |
| 面板状态 | 展示当前任务、进度/耗时、成功或失败；批量与单条共用队列 |
| 与列表同步 | 提炼过程更新会话项状态（如 analyzing） |

---

## 13. 其他包与占位

| 项 | 状态 |
|----|------|
| `packages/shared` | 共享类型与常量（如卡片类型文案等） |
| `packages/mcp-server` | **仅占位**：进程启动即退出并提示未实现，对应后续版本 |
| `packages/sidecar` | 正式提炼逻辑与构建产物（桌面 `build:tauri` 会构建 sidecar） |

---

## 14. 配置与同步行为（实现现状）

| 配置项 | v0.1 行为 |
|--------|-----------|
| `sync.mode` | 配置结构支持 `manual` / `ask` / `auto`；**实现上为手动同步**，自动/定时未作为常驻后台任务启用（以 `config.rs` 注释为准） |

---

## 15. 小结

v0.1 已形成 **「采集 → 列表 → 提炼 → 笔记/回放 → 搜索/知识库」** 的本地闭环；**多 IDE 全量接入、MCP、自动同步、导出** 等仍属后续迭代。若需验收，建议在真实 Claude Code / Cursor 数据与有效 API 或 CLI 环境下跑通同步与单条/批量提炼全流程。
