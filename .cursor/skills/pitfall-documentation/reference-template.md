# 单条踩坑条目 — Markdown 模板

将下列块作为新小节（`###`）插入现有文档，或作为新文件正文骨架。标题级数可与仓库内已有 `docs/踩坑/*.md` 对齐。

```markdown
## <简短标题>（可选：关联模块名）

### 现象

- 用户或日志中可见的具体表现（错误信息、截图要点、复现步骤摘要）。

### 根因

- 技术层面的原因（代码路径、配置、环境、时序等）。可附 ASCII 示意。

### 解决方法

- 已采用的修复步骤或配置变更；关键文件/函数用反引号标出。

### 后续如何避免（可选）

- 规范、测试、静态检查、文档约定、监控告警等可执行建议；若无则写「暂无，需人工注意 <点>」或省略本小节。

### 相关位置（可选）

- `path/to/file` — 一句话说明
```

## 文件命名建议

- **以中文为主**、`-` 分段；技术专名可保留英文（`UnoCSS`、`CLI`、`JSON` 等）。示例：`桌面应用-会话提炼-JSON解析与标签技术栈.md`。
- 完整约定与索引见仓库 [`docs/踩坑/README.md`](../../../docs/踩坑/README.md)。
- 同一主题持续踩坑：优先 **合并进已有文件** 为新 `##` / `###` 小节，避免碎片化。
